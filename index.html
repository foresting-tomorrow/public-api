HTTP Status Code Summary

Code Summary Description
200 OK Everything worked as expected.
201 Created The request has been fulfilled and resulted in a new resource being created.
400 Bad Request The server cannot or will not process the request due to an apparent client error.
401 Unauthorized Unauthorized access to the resource.
403 Forbidden The server understood the request, but is refusing to fulfill it.
404 Not Found The requested resource could not be found.
405 Method Not Allowed A request method is not supported for the requested resource.
429 Too Many Requests The user has sent too many requests in a given amount of time.
500 Internal Server Error The server has encountered a situation it doesn't know how to handle.
503 Service Unavailable The server is not ready to handle the request.

For each resource only the status codes in the 200-299 are documented in the OpenAPI spec. For all other status codes the data object will be a string.

Disclaimer

Some operations, make use of objects in query parameters. Beware that some rest clients have issues when importing a spec containing these. As an example postman will unfold the the object, such that each field becomes a query parameter.

As an example a query parameter object like this:

query='{ "key1": "value1", "key2": "value2" }'

Will be unfolded to:

key1=value1&key2=value2

When imported into postman.
